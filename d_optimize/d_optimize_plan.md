# RW 配置 D 最適化 — 実装計画

## 1. 問題の概要

`position_transform.py` では 6 配置の変換行列 T_i をあらかじめ手動で決めていた。
ここでは **D 最適計画の基準** を用いて、伝達行列同定に最も適した RW 配置（位置・スピン軸方向）を自動決定する。

---

## 2. 物理モデルと U_mat の構造

### 変換行列（`make_T` の再利用）

位置 r、スピン軸方向 d（単位ベクトル）に対して：

```
T(r, d) = [ R          0 ]   (6×6)
           [ S(r)@R    R ]
```

ここで R = rotation_to_align(d)（ローカル x 軸を d に合わせる回転行列）。

### U_mat = [T_1 u | T_2 u | ... | T_6 u] における u の扱い

同定では以下の線形系を解く：

```
Y_mat = G @ U_mat
U_mat[:,i] = T_i @ u(ω)    ← u(ω) は実測前には未知
```

`u(ω)` は RW 本体の物性（質量不釣り合い等）から決まる力学ベクトルであり、配置とは独立に存在する。
**測定前には u の方向は不明である。**

#### ベースライン仮定（静的不釣り合い優位モデル）

RW の主要外乱は **スピン軸方向の力（静的不釣り合い成分）** であるため、
ローカル座標での u を次のように近似する：

```
u_local ≈ F · [1, 0, 0, 0, 0, 0]^T    （スピン軸 = ローカル x 軸）
```

この仮定のもとで：

```
T(r_i, d_i) @ u_local = F · [ d_i        ]
                              [ r_i × d_i ]
```

U_mat の各列がそれぞれの配置のランチベクトル（wrench vector）になる。
F はスカラー倍にすぎず `log|det|` には影響しないため、配置最適化から除外できる。

#### u が未知であることの影響

- u の **方向が周波数によらず一定**（振幅だけ変化 → PSD変化）であれば、上記仮定の範囲で最適化は成立する。
- 高次高調波など u の **方向が変わる**場合は、複数の u_n を想定して以下のような周波数平均基準に拡張する：

```
maximize  Σ_n  w_n · log |det(U_mat(u_n))|
          w_n = PSD(ω_n) / Σ PSD  （PSD重み）
```

まずはベースライン仮定で実装し、測定後に u_n を代入して再評価・再最適化する。

---

## 3. D 最適基準の定式化

### D 最適目的関数

情報行列 M = U_mat @ U_mat^T (6×6) として：

```
maximize  f = log |det(U_mat)|
```

U_mat が 6×6 のとき det(M) = det(U_mat)^2 なので `log|det(U_mat)|` の最大化と等価。
これが参考記事の「det(X^T X) を最大化する」D 最適基準に対応する。

## 4. 設計変数

| 変数 | 内容 | 次元 |
|------|------|------|
| `r_1 = (0,0,0)` | 位置 1（固定） | — |
| `r_2 = (1,0,0)` | 位置 2（固定） | — |
| `r_3 = (x,y,z)` | 位置 3（自由） | 3 |
| `d_11, d_12` | 位置 1 の RW1, RW2 スピン軸 | 2×2 = 4 |
| `d_21, d_22` | 位置 2 の RW1, RW2 スピン軸 | 2×2 = 4 |
| `d_31, d_32` | 位置 3 の RW1, RW2 スピン軸 | 2×2 = 4 |

**合計 15 自由変数**

### 単位ベクトルの球面パラメータ化

|d|=1 を制約として陽に扱わず、角度変数で自動的に満たす：

```
d(θ, φ) = (sin θ cos φ,  sin θ sin φ,  cos θ)
```

θ ∈ [0, π]、φ ∈ [-π/2, π/2] とすることで **d_x = sin θ cos φ ≥ 0** を保証。
→ 6 つのスピン軸を (θ_k, φ_k) の 12 変数で表現。

### スピン方向の符号不定性について

d と -d はランチベクトルの符号を反転させるだけ（= U_mat の 1 列の符号が変わるだけ）なので、
`|det(U_mat)|` に対して全く同等である。
球面パラメータ化で **φ ∈ [-π/2, π/2]（= d_x ≥ 0）** に制限することで
各スピン軸ごとの 2 重解を一意に除去する。

### r と d のスケールについて

- **d**: |d|=1 に正規化済み → 力ブロック（U_mat の上 3 行）の各列の大きさは常に 1。スケール問題なし。
- **r**: トルクブロック（U_mat の下 3 行）の各列 = r × d の大きさは |r|sin(angle)。
  r の大きさが直接トルクブロックの値を決める。
  **r の物理的スケール（衛星サイズ [m]）は設計制約として陽に設定すること**（次節参照）。
  位置の制約を適切に設ければ、数値的スケール問題は実用上問題にならない。

## 5. 最適化問題のまとめ

```
maximize   log |det(U_mat(x, y, z, θ_1, φ_1, ..., θ_6, φ_6))|

変数上下界:
  r_3: 0 ≤ x ≤ 2,  -1 ≤ y ≤ 1,  -1 ≤ z ≤ 1
  角度: 0 ≤ θ_k ≤ π,  -π/2 ≤ φ_k ≤ π/2   （d_x ≥ 0 を保証）
```

## 8. 懸念点・注意事項

| 項目 | 内容 |
|------|------|
| u の不確かさ | ベースライン仮定（静的不釣り合い優位）で最適化。測定後に u_n を代入し再評価する |
| 周波数重み | u の方向が変わる高次高調波を含む場合は PSD 重み付き和に拡張 |
| 局所最適 | det は多峰性のため differential_evolution で大域探索必須 |
| 退化リスク | 同一位置の 2 RW が同方向 d → 2 列が比例してランク落ち。初期値の多様性で回避 |
| r スケール | 衛星サイズに合わせて位置上下界を設定すること（r が小さすぎるとトルクブロックが消える） |
